Offline-ATM ⚡️ WEB INSTALLER

Use with esp32dev microcontroller

1- Flash firmware

  • Connect your ATM to the USB port of the esp32dev board
  • Press 🔥 Flash - select the connection and follow screen instructions
Your browser does not support installing things on ESP devices. Use Google Chrome or Microsoft Edge. Ah snap, you are not allowed to use this on HTTP!

Note: The configuration data can be deleted using "Erase device." Otherwise, it will be retained even with the new firmware. LOGS & CONSOLE is not needed.

⏳ After flashing: Wait at least 15 seconds until the e-paper screen updates.
Then go to 3- Load config values and press 🔌 Connect.

2- Prepare connection

When you flash a new ESP32 or use "Erase device." the ATM automatically starts in configuration mode.

💡 Config mode indicator: The LED button blinks slowly (~1× per second) when the ATM is in configuration mode. Solid LED = normal operation.

→ The ATM is now ready to receive configuration data. ⚙️

If parameters have already been stored and need to be changed, you can also open CONF mode retrospectively:

  • Press and hold BOOT button for 5 seconds.
  • Press the LED button once briefly and then hold it down for at least 5 seconds.

Note: The connection will remain active for 3 minutes (180 seconds) before it is automatically disconnected. If data is already available, you can terminate the connection prematurely by pressing the BOOT button or pressing the external button.

3- Load config values

  • Press 🔌 Connect - select the connection and connect.
  • Check whether the ATM is in config mode.⚙️✅
  • If values are already available, press 📖 Read Config.
  • Define your configuration.📝 At least the Device String.
  • Press 🔥 Write Config to write the configuration.
  • Press 🔄 Restart to restart while staying connected.
  • Press 🔌 Disconnect or refresh website to close connection.

Generate the device string in LNbits (fossa extension or lnurldevice extension) and paste it here.

Troubleshoot

Connection problems:

  • Check that the USB cable is plugged directly into the USB port of the ESP32 and not into the USB port of a power supply.
  • Check/replace the cable. There are USB cables that have no data lines and are only for power supply.
  • Double connection: Check whether a connection already exists via another browser window. Close one window.
  • A website refresh with CTRL+F5 or CTRL+SHIFT+R can also help.
  • If nothing else helps, do a complete restart by erasing all data on the ESP32. Use https://espressif.github.io/esptool-js/. → Press: Connect > Erase Flash and reflash the ATM.

Further note:

Some ESP32 boards must first be put into bootloader mode before flashing or transferring data. Try these:

  • Connect the board via the USB cable
  • Press and hold the BOOT button
  • While still pressing the BOOT button, press RESET (EN)
  • Release the RESET button
  • Release the BOOT button
  • Go to step 1
